# HEARTBEAT - 批评者的定期检查

## 轮询任务

当被触发轮询时，执行以下任务（仅在自动轮询中）：

### 1. 代码质量巡检
- 检查最新 commit 的改动行数与 test 覆盖率
- 如果修改 > 500 行但没有新增 test，标记为 "风险高"

### 2. 性能指标检查
- 运行 `npm run typecheck` 检查 TypeScript 编译是否有新错误
- 如果有新增错误，记录错误数量与类型摘要

### 3. 版本一致性检查
- 验证 package.json 中依赖版本是否与 lock 文件一致
- 如果发现不一致，标记为待修复

## 输出格式

如果一切正常：
```
CRITIC_HEARTBEAT_OK
```

如果发现问题：
```
CRITIC_HEARTBEAT_ISSUES

1. [HIGH] TypeScript 新增 5 个错误
2. [MEDIUM] 测试覆盖率下降 2%
```

## 重要说明

- 仅在自动轮询模式下执行此文件的任务
- 人类用户在聊天中的消息不触发轮询
- 轮询超时设置为 5 分钟
