# HEARTBEAT - 规划师的定期检查

## 轮询任务

当被触发轮询时，执行以下任务（仅在自动轮询中）：

### 1. 任务进展盘检
- 检查 `memory/TASK_QUEUE.md` 中所有任务的状态变化
- 计算已完成 / 进行中 / 待办的任务数
- 如果有任务超过预计时间 50% 以上，标记为"需要重新评估"

### 2. 关键阻塞项排查
- 读取 `memory/STATUS.md` 中的"阻塞项"部分
- 检查这些阻塞项是否是新增的（与上次轮询对比）
- 如果有新阻塞项，需要判断是否需要调整当前计划优先级

### 3. 决策一致性检查
- 核对 `memory/DECISIONS.md` 中的决策是否被正确执行
- 如果代码/实现与决策不符，标记为"执行偏差"

## 输出格式

如果一切正常：
```
PLANNER_HEARTBEAT_OK
```

如果发现问题：
```
PLANNER_HEARTBEAT_ISSUES

1. [HIGH] 任务超期
   - 任务：X
   - 预计完成：N 小时前
   - 建议：重新评估或调整优先级

2. [MEDIUM] 新阻塞项出现
   - 阻塞项：Y
   - 影响任务：Z
   - 建议：需要 Executor 反馈可行方案
```

## 重要说明

- 仅在自动轮询模式下执行此文件的任务
- 人类用户消息不触发轮询
- 每次轮询超时设置为 5 分钟
- 轮询频率：每 30 分钟一次（可选）
