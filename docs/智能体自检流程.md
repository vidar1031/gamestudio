# 智能体自检流程

本文档定义 HermesManager 在左脑启动前执行的显式自检流程。

## 目标

左脑配置不是静态表单，而是启动前置流程。

用户点击“自检”后，系统必须把当前左脑配置转成一次真实检查，并把结果清楚展示出来。只有全部通过，左脑和 Hermes 才允许启动。

## 自检步骤

1. 检测 Hermes 是否存在
   - 检查 Hermes gateway 脚本和 Python 运行入口是否可调用
   - 成功时显示：Hermes OK
   - 失败时显示红色错误，并阻止启动

2. 检测 Hermes 配置目录和文件
   - 检查左脑配置中的记忆、agent、skill 等路径
   - 源文件包括：ai/AGENTS.md、ai/USER.md、ai/MEMORY.md、ai/memory/LONG_TASKS.md、ai/memory/STATUS.md、ai/memory/TASK_QUEUE.md、ai/memory/DECISIONS.md、技能根目录、SKILL.md
   - 自检时只检查左脑配置里保存的 ai 路径，不再生成根目录兼容文件
   - 每项都单独显示 OK 或错误原因

3. 检测本地模型服务 OMLX 是否运行
   - 请求 http://127.0.0.1:18888/v1/models
   - 成功时显示：OMLX OK
   - 若可读到模型列表，则可自动选中当前可用模型
   - 如果没有检测到模型，则显示：请选择模型

4. 主动推理检测
   - 使用当前左脑 provider、baseUrl、model 执行一次最小推理探测
   - 该步骤不是纯路径检查，而是真实调用模型接口
   - 推理失败时以红色日志输出错误摘要

## 启动约束

只有下列条件同时满足，启动按钮才可点击：

- 左脑配置已保存
- Hermes 安装检测通过
- 配置文件与目录全部存在
- OMLX 服务连通
- 已选中有效模型
- 主动推理检测通过
- 右脑保持禁用

## 读写边界

- ai/ 和 ai/memory/ 是 GameStudio 的源文件位置
- Hermes 原生内建记忆不直接写回 ai/memory
- manager 负责保存并校验 Hermes 左脑配置，Hermes 启动时只读取已保存的 ai 路径
- Hermes 自己的内建记忆写入位置，仍以 Hermes 自身 memories 机制为准
- 如果本轮任务已经改变项目状态、短任务、稳定规则或长任务主线，实际运行后还应确认对应记忆文件会被同步，而不是只看到聊天输出更新

## 日志要求

自检和启动都必须写入日志区，至少包含：

- 自检开始
- 每个步骤的 OK 或 ERROR
- 模型准备完成
- Hermes 启动开始
- Hermes 启动完成或失败

错误日志使用红色展示，方便用户直接定位阻塞项；所有自检和启动日志统一进入顶部日志窗口。
