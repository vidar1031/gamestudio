# UI 自我约束（执行规范）

目标：保证 `gamestudio` 的界面一致、可读、可操作，不再出现“样式拼贴/临时补丁”问题。

## 1. 视觉约束
- 统一深色基底与单一强调色，不在同一页面混用多套风格。
- 组件状态只允许三种层级：默认 / hover / active，不增加额外花哨动画。
- 首屏与设置页优先信息清晰，不做装饰性大块渐变卡片。

## 2. 布局约束
- 主界面固定三栏语义：列表 / 主操作区 / 说明。
- 设置页单页直出，不再使用“参数/检测”分标签切换。
- 避免嵌套滚动视觉；滚动能力保留但默认隐藏滚动条显示。

## 3. 文案约束
- 只保留用户可执行信息，不展示阶段性研发术语（如 P0/P1/P2）。
- 说明文案必须与当前操作直接相关，避免历史背景与内部术语。
- Provider/Model 展示统一格式：`provider / model`，未启用显示 `无`。

## 4. 交互约束
- 所有“测试连接”都在当前配置区就地执行，不跳转到二级页面。
- 弹窗顶部仅保留关键操作：`保存并应用`、`关闭`。
- 未配置 Provider 时，动作必须给出明确阻断提示，而不是静默失败。

## 5. 发布前检查
- 检查是否出现多余滚动条、溢出遮挡、按钮不可见。
- 检查首页、设置页、AI 创建弹窗在 1366x768 与 1920x1080 下可用。
- 检查文案是否包含研发内部标记词（P0/P1/实验/临时）。
