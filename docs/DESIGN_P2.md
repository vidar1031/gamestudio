# P2 设计方向（发布优先）

P2 目标：把当前“可用制作工具”升级为“可稳定发布的 H5 生产工具”。

> 说明：本版 P2 已按产品要求去掉“多人协作”，优先级最高的是“生成可发布包（HTML H5）”。

## 1. 首要目标（必须先完成）

## 1.1 生成可发布包（H5）
- 一键导出可分发的发布包（zip）
- 包内至少包含：
  - `dist/index.html`
  - `dist/story.json`
  - `dist/project.json`
  - `dist/game.manifest.json`
  - `dist/assets/*`
- 导出结果必须可本地解压后直接运行（静态资源相对路径有效）

## 1.2 发布前校验
- 结构校验：节点可达、结局可达、起点存在
- 资源校验：引用资源存在、URI 可解析
- 运行校验：manifest 与 entry 一致

## 2. P2 功能范围（去掉多人协作后）

1. 资产与风格工业化
- 世界观/角色一致性评分
- LoRA 从“预留配置”升级为“生效参数”
- 工作流模板（风格、模型、尺寸、采样器）可复用

2. 任务调度中心
- 批量提示词与批量出图队列化
- 支持暂停、取消、继续未完成、失败自动重试策略
- 长任务后台执行与恢复

3. 质量评估与自动验收
- 文本结构、分支逻辑、画面一致性自动评分
- 不达标自动打回并给修复建议

4. 发布链路衔接
- 对接 app_system 的导入协议
- 输出标准化发布元信息（版本、hash、构建时间、依赖模型）

## 3. P2 里程碑建议

## P2-M1（发布闭环）
- 完成“导出发布包(H5)”与发布前校验
- 产出固定包结构与 manifest 版本字段

## P2-M2（稳定生产）
- 任务调度中心（暂停/取消/续跑）
- 错误恢复策略与可观测性补强

## P2-M3（质量驱动）
- 自动评分与验收门禁
- LoRA/工作流模板正式生效
