# LONG_TASKS - 长任务主线

## LT-01 控制驱动的故事项目创建

- 目标：让 control 调用 Hermes 在 `storage/projects` 下创建标准项目骨架
- 期望产物：项目目录、基础元数据、初始 story / scripts / blueprint 文件
- 完成标准：用户能通过 control 发起创建任务，并得到可验证的项目目录与初始化结果

## LT-02 项目配置与模板装配

- 目标：让 control / Hermes 能配置项目风格、模板、模型与生成参数
- 期望产物：可重复使用的配置输入结构与项目内持久化配置
- 完成标准：项目配置不依赖手工零散编辑，能够稳定复用

## LT-03 故事文本与结构生成

- 目标：从 brief 生成 story、scripts、blueprint 等结构化内容
- 期望产物：明确的输入、生成步骤、验证规则与回填路径
- 完成标准：至少一条从 brief 到可读结构文件的端到端链路可重复运行

## LT-04 图片与资产生成回填

- 目标：让 Hermes 驱动图片与资产生成，并回填到项目资源引用中
- 期望产物：图片生成任务链、资产入库规则、引用回填规则
- 完成标准：至少一类图片生成任务能稳定落到项目目录并被项目引用

## LT-05 项目验证与交付

- 目标：对生成结果做结构校验、资源校验、导出与验收
- 期望产物：验证清单、失败结论格式、导出路径说明
- 完成标准：control 可以给出明确的 PASS / FAIL 与证据摘要

## LT-06 运行时记忆自维护闭环

- 目标：让 control / Hermes 在多步骤任务后自动维护 `STATUS.md`、`TASK_QUEUE.md`、`DECISIONS.md`、`LONG_TASKS.md` 与当日日志
- 期望产物：planner 能生成记忆同步步骤，runtime 在审核通过后落盘最新状态
- 完成标准：下一轮端到端测试中，上述记忆文件能随着任务推进自动保持最新且互相一致
