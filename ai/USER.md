# USER - Collaboration Preferences

## 默认协作方式

- 默认用中文沟通。
- 优先先执行、后汇报，不先给长篇方案等待确认。
- 汇报优先包含：做了什么、是否验证通过、下一步是什么。
- 只有在高风险、强歧义或需要用户拍板时才停下来问。

## 当前项目期望

- 重点目标是让 `control` 调度 `Hermes`，完成故事项目的创建、配置、生成与验证。
- 项目理解必须落到文件记忆，不依赖短期聊天印象。
- 对 stories / 项目扫描类问题，需要给出明确的目录证据，而不是泛化描述。

## 工程偏好

- 本地优先：优先使用本地模型、本地服务和本地可验证链路。
- 小步快跑：一次只做一个清晰动作批次，改后立即验证。
- 不做无关清理：只改当前任务需要的部分。
- 删除文件前要先确认它们确实过时或无效；本次用户已明确授权删除 `ai/` 中的旧文件和旧记录。

## 记忆维护规则

- 长任务方向变化：更新 `ai/MEMORY.md` 或 `ai/memory/LONG_TASKS.md`
- 短任务变化：更新 `ai/memory/TASK_QUEUE.md`
- 当前状态变化：更新 `ai/memory/STATUS.md`
- 生效中的新规则：更新 `ai/memory/DECISIONS.md`
- 当日关键进展：更新最新 `ai/memory/YYYY-MM-DD.md`

## 不希望出现的行为

- 不要引用已经废弃的旧协作流程或历史自动化设定。
- 不要把未扫描到的目录内容说成已经扫描过。
- 不要在没有验证的情况下声称“已经完成”。
