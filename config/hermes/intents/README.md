# Hermes Intent Definitions

This folder holds **declarative** intent definitions that extend the
control-server reasoning pipeline without changing JavaScript code.

- `*.json` in this folder is auto-loaded at control-server startup.
- `_inbox/*.json` is generated by the proposal pipeline (see
  `apps/control-server/src/server/intents/proposals.js`) and waits for
  promotion via `POST /api/control/skill-proposals/:id/promote`.

JSON intents are pure data — they cannot execute arbitrary code. The
JSON is validated by `compileJsonIntent` before it is ever consulted by
the live pipeline. See the inline schema docs in `jsonIntent.js`.

## Why this exists

`apps/control-server/src/server/controlServerCore.js` previously grew a
new `if (isXxxPrompt) ...` arm every time a new task type appeared. The
intent registry (`apps/control-server/src/server/intents/registry.js`)
and this folder replace that pattern with:

1. **Builtin intents** — registered programmatically inside
   `controlServerCore.js` for legacy detectors that already exist.
2. **JSON intents** — operator-authored declarative rules under this
   folder.
3. **Proposals** — agent-suggested rules under `_inbox/`, gated by
   human (or higher-trust agent) approval before they go live.

This gives the GameStudio agent a path to refine its own reasoning
quality gates without rewriting Node code or unblocking unsafe code
paths.
